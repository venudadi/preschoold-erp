COMMON TESTING SCENARIOS
================================================================================

SCENARIO 1: USER MANAGEMENT
  1. Login as super admin
  2. Navigate to "User Management"
  3. Click "Add User"
  4. Fill in details and assign role
  5. Test login with the new user

SCENARIO 2: PASSWORD RESET (SUPER ADMIN FEATURE)
  1. Login as super admin
  2. Go to User Management
  3. Click "Reset Password" on any user
  4. Enter new password
  5. Test login with new password

SCENARIO 3: CLASSROOM WORKFLOW
  1. Login as teacher
  2. Create a lesson plan
  3. Add an assignment
  4. Record student observations
  5. Upload to digital portfolio

SCENARIO 4: PARENT COMMUNICATION
  1. Login as teacher
  2. Navigate to Messages
  3. Start conversation with parent
  4. Login as parent
  5. View and reply to message

================================================================================
TROUBLESHOOTING
================================================================================

🔴 LOGIN FAILS WITH 500 ERROR
  → Check backend is running on port 5001
  → Verify database connection
  → Check backend console for errors

🔴 LOGIN FAILS WITH 401/403 ERROR
  → Verify you're using correct credentials
  → Check TEST_CREDENTIALS.txt for latest passwords
  → All test passwords are: Test@123

🔴 BLANK PAGE OR MODULE NOT LOADING
  → Check browser console (F12)
  → Look for red errors
  → Verify all database tables exist

🔴 CORS ERRORS
  → Backend should allow ports 5173, 5174, 5175
  → Restart backend server
  → Clear browser cache

🔴 DATABASE ERRORS
  → Run: cd backend && node check_and_run_migrations.js
  → Verify MySQL is running
  → Check database credentials in backend/.env
